## 摘要

**【目的/意义】** 《数据安全法》《个人信息保护法》实施后，政府数据开放平台面临"数据开放"与"隐私保护"的双重张力。国家数据局正在制定"数据流通匿名化处理评价标准"，但尚缺乏针对政府数据开放平台数据集的匿名化质量评估实证研究。本文构建基于匿名化处理标准的平台数据集质量评估框架，为平台匿名化处理实践提供可操作的评估工具。

**【方法/过程】** 以冉从敬教授参与制定的数据流通匿名化处理评价标准为理论基础，构建包含"识别风险、关联风险、推断风险"三个维度的匿名化质量评估指标体系。选取我国23个省级政府数据开放平台，采用内容分析法对平台数据集中的个人信息字段进行编码，结合专家评分法评估各平台数据集的匿名化处理质量，并与数据集利用绩效进行关联分析。

**【结果/结论】** 研究发现：（1）23个平台中仅35%的数据集采用了规范的匿名化处理，65%存在不同程度的个人信息泄露风险；（2）匿名化处理质量与数据集下载量呈倒U型关系——适度匿名化（保留数据可用性）促进利用，过度匿名化（破坏数据结构）抑制利用；（3）平台间的匿名化质量差异显著，东部省份优于中西部省份；（4）当前平台匿名化处理存在"一刀切"倾向，缺乏差异化的分级分类处理策略。研究建议建立"数据集敏感度分级×匿名化强度匹配"的处理矩阵，为《数据流通匿名化处理评价标准》在政府数据开放场景下的落地实施提供实证支撑。

**关键词**：政府数据开放；匿名化处理；数据流通标准；数据集质量；个人信息保护；数据安全法

## 一、引言

### 1.1 研究背景

政府数据开放是推进国家治理体系和治理能力现代化的重要举措。截至2025年底，我国已有31个省级行政区建立了政府数据开放平台，累计开放数据集超过35万个，涵盖了交通、环境、医疗、教育、经济等数十个领域。然而，随着数据开放规模的扩大，一个深层矛盾日益凸显：**如何在保障数据可用性的前提下，有效防范个人信息泄露风险？**

2021年《个人信息保护法》的实施，对政府数据开放平台提出了明确的合规要求。该法第五十一条规定，个人信息处理者应当对个人信息实行分类管理，并采取相应的加密、去标识化等安全技术措施。2021年《数据安全法》第二十七条进一步明确，开展数据处理活动应当加强风险监测，发现数据安全缺陷、漏洞等风险时，应当立即采取补救措施。这些法律框架的确立，标志着政府数据开放从"无条件开放"走向"有条件开放"的新阶段。

在此背景下，**匿名化处理**成为政府数据开放平台面临的核心技术-制度难题。一方面，过度匿名化可能导致数据丧失分析价值，抑制数据利用者的使用意愿；另一方面，匿名化不足则可能造成个人信息泄露，引发法律风险和公众信任危机。更为复杂的是，不同类型的数据集（如人口统计、医疗记录、交通轨迹）具有不同的敏感度，需要差异化的匿名化处理策略。然而，当前实践中，各平台普遍缺乏统一的匿名化处理标准和质量评估工具，处理水平参差不齐。

2024年，国家数据局启动"数据流通匿名化处理评价标准"制定工作，旨在建立全国统一的匿名化处理质量评价体系。冉从敬教授团队承担了该标准的研制任务。本研究正是在这一政策背景下，以该标准的核心框架为理论基础，针对政府数据开放平台的数据集进行匿名化质量评估，为标准在政府数据开放场景下的落地实施提供实证支撑。

### 1.2 问题提出

基于上述背景，本研究聚焦以下核心问题：

**RQ1**：当前我国省级政府数据开放平台的数据集匿名化处理处于什么水平？存在哪些共性问题？

**RQ2**：匿名化处理质量如何影响数据集的利用绩效？两者之间存在什么样的关联机制？

**RQ3**：如何构建适用于政府数据开放平台的匿名化处理质量分级分类框架？

### 1.3 研究意义

**理论意义**：本研究将数据流通匿名化处理标准从抽象的制度文本转化为可操作的评估工具，填补了"标准-实践"之间的评估空白。同时，通过引入"数据可用性-隐私保护"权衡视角，丰富了政府数据开放评价的理论框架。

**实践意义**：研究直接服务于国家数据局正在制定的匿名化处理评价标准，为标准的场景化应用提供实证依据。对平台管理者而言，评估结果可直接用于识别匿名化处理薄弱环节，指导改进实践。

## 二、文献综述

### 2.1 政府数据开放中的个人信息保护研究

政府数据开放中的个人信息保护是近年来学界关注的热点议题。Janssen等（2012）最早指出，开放数据运动面临"开放与隐私"的根本张力。Zuiderwijk等（2014）通过跨国比较发现，各国政府在数据开放中对个人信息保护的处理方式差异显著，缺乏统一标准。

在国内，黄如花等（2019）对我国政府数据开放平台进行了系统性评估，发现多数平台未建立个人信息审查机制。夏义堃等（2020）指出，政府数据开放中的"去标识化"处理存在标准不一、技术手段落后等问题。张晓娟等（2021）提出了"分类分级开放"理念，主张根据数据敏感度和利用价值建立差异化开放策略。

然而，现有研究多停留在理念层面或个案分析，缺乏大规模平台的系统性实证评估，也未能与正在制定的国家标准进行对接。

### 2.2 匿名化处理技术与标准研究

匿名化处理技术经历了从简单脱敏到差分隐私的演进。Sweeney（2002）提出的k-匿名模型是早期的经典方法，要求数据集中每条记录至少与k-1条其他记录不可区分。Dwork（2006）提出的差分隐私则通过添加可控噪声实现隐私保护，成为当前学术界的主流方法。

在标准制定方面，国际标准化组织（ISO）发布了ISO/IEC 20889:2018《隐私增强数据去标识化框架》，为数据去标识化提供了技术指南。欧盟GDPR将"匿名化数据"排除在个人数据定义之外，但对匿名化标准未作具体规定。美国NIST发布了《去标识化指南》（NISTIR 8053），提供了去标识化的风险评估框架。

在国内，全国信息安全标准化技术委员会发布了GB/T 37964-2019《个人信息去标识化指南》，但主要针对企业数据处理场景，对政府数据开放场景的适用性有限。2024年，国家数据局启动"数据流通匿名化处理评价标准"制定工作，标志着我国匿名化处理标准化进入新阶段。

### 2.3 数据质量评估研究

数据质量评估是信息资源管理领域的经典议题。Wang和Strong（1996）提出了包含内在质量、情境质量、表达质量和可访问质量四个维度的数据质量框架。Batini等（2009）进一步将数据质量维度细化为准确性、完整性、一致性、及时性、可用性等。

在政府数据开放领域，数据质量评估已成为平台评价的核心内容。Veljkovic等（2010）提出了开放政府数据质量评估的五维度框架（准确性、完整性、及时性、可访问性、可用性）。国内学者如黄如花等（2019）、张会平等（2020）均构建了面向政府数据开放平台的质量评估指标体系。

然而，现有数据质量评估框架普遍未将"匿名化处理质量"纳入考量。在《个人信息保护法》实施背景下，匿名化质量已成为数据集质量不可或缺的维度。

### 2.4 研究述评

综合现有文献，可以发现三个研究空白：

**第一**，缺乏与国家标准对接的匿名化质量评估实证研究。现有研究多停留在理论探讨或个案分析，未能与正在制定的"数据流通匿名化处理评价标准"进行对接。

**第二**，匿名化质量与数据利用绩效的关联机制尚不明确。现有研究将匿名化和数据利用视为两个独立议题，缺乏对两者关联的系统性考察。

**第三**，缺乏大规模平台的匿名化处理现状评估。现有评估多基于少量平台的案例分析，缺乏覆盖全国省级平台的系统性评估。

本研究旨在填补上述空白，构建基于国家标准的匿名化质量评估框架，并开展23个省级平台的实证评估。

## 三、理论框架与分析模型

### 3.1 匿名化处理质量的理论基础

本研究以冉从敬教授团队参与制定的"数据流通匿名化处理评价标准"为核心理论基础。该标准将匿名化处理质量定义为"数据经匿名化处理后，在保障个人信息不被识别的前提下，最大限度保留数据利用价值的能力"。基于该定义，标准构建了三个核心评估维度：

**识别风险（Identification Risk）**：指攻击者通过直接标识符（如姓名、身份证号）或准标识符（如性别、年龄、邮编的组合）识别特定个人身份的概率。识别风险是匿名化处理的首要目标，也是匿名化质量的基础门槛。

**关联风险（Linkage Risk）**：指攻击者通过将匿名化数据集与其他外部数据集进行关联，从而推断个人身份或敏感信息的风险。关联风险评估需要假设攻击者可能获取的外部数据集，具有较强的场景依赖性。

**推断风险（Inference Risk）**：指攻击者通过分析匿名化数据集中的统计规律，推断出特定个人或群体的敏感属性（如疾病、收入、政治倾向）的风险。推断风险在高度聚合的数据集中尤为突出。

### 3.2 "数据可用性-隐私保护"权衡模型

本研究引入经济学中的"权衡分析"视角，构建"数据可用性-隐私保护"权衡模型（Trade-off Model between Data Utility and Privacy Protection，简称TU-PP模型）。

模型的基本假设是：匿名化处理强度（Anonymization Intensity，AI）与数据可用性（Data Utility，DU）呈负相关关系，与隐私保护水平（Privacy Protection Level，PPL）呈正相关关系。存在一个最优匿名化强度区间，使得数据利用绩效（Data Use Performance，DUP）最大化。

用公式表示：

```
DUP = f(DU, PPL) = α·DU + β·PPL - γ·|AI - AI*|
```

其中，AI*为最优匿名化强度，α、β、γ为权重系数。当AI < AI*时，隐私保护不足导致利用者信任缺失，抑制DUP；当AI > AI*时，数据可用性过低导致利用价值下降，同样抑制DUP。

### 3.3 分析框架

基于上述理论基础，本研究构建如下分析框架：

```
【输入层】
  23个省级平台数据集样本
       ↓
【评估层】
  识别风险评估 → 关联风险评估 → 推断风险评估
       ↓
【质量分级层】
  高风险 / 中风险 / 低风险 / 合规
       ↓
【关联分析层】
  匿名化质量 ↔ 下载量 / 利用案例 / API调用
       ↓
【输出层】
  分级分类处理策略建议
```

## 四、研究设计

### 4.1 研究样本

本研究选取我国23个省级政府数据开放平台作为研究样本。样本选择遵循以下原则：（1）平台运营时间超过2年，形成稳定的数据集积累；（2）平台公开的数据集目录可获取；（3）平台覆盖东、中、西部不同经济发展水平的省份。样本涵盖北京、上海、天津、重庆4个直辖市，广东、浙江、江苏、山东、福建、海南等东部省份，湖北、湖南、河南、安徽、江西等中部省份，以及四川、贵州、云南、广西、辽宁、吉林、内蒙古、山西等西部和东北部省份。

在每个平台上，采用分层随机抽样方法抽取数据集样本。第一层按数据集主题（人口、医疗、交通、环境、经济、教育、文化、其他）进行分层，第二层在各主题层内随机抽取。每个平台抽取30个数据集，总样本量为690个数据集。

### 4.2 评估指标体系

基于"数据流通匿名化处理评价标准"的核心框架，本研究构建了适用于政府数据开放平台场景的三级评估指标体系：

**一级指标：匿名化处理质量指数（AQI）**

**二级指标与三级指标**：

| 二级指标 | 三级指标 | 评估方法 | 权重 |
|:---|:---|:---|:---|
| **识别风险（IR）** | 直接标识符移除率 | 字段审查法 | 0.30 |
| | 准标识符泛化程度 | 字段审查法 | 0.20 |
| | 特殊群体识别风险 | 内容审查法 | 0.10 |
| **关联风险（LR）** | 准标识符组合唯一性 | 统计检验法 | 0.15 |
| | 外部数据集关联可能性 | 场景假设法 | 0.10 |
| **推断风险（IFR）** | 统计推断风险 | 敏感性分析法 | 0.10 |
| | 属性推断风险 | 内容审查法 | 0.05 |

**评分规则**：
- 每个三级指标按5级评分：1=极差，2=较差，3=一般，4=良好，5=优秀
- 二级指标得分 = 三级指标加权平均
- AQI = IR×0.6 + LR×0.25 + IFR×0.15
- AQI ≥ 4.0为"合规"，3.0-4.0为"低风险"，2.0-3.0为"中风险"，<2.0为"高风险"

### 4.3 数据收集方法

**（1）数据集字段审查**：由两名编码员独立对690个数据集的数据字典、字段说明进行审查，识别直接标识符（姓名、身份证号、手机号等）和准标识符（性别、年龄、职业、邮编等）。采用Cohen's Kappa系数检验编码一致性，Kappa>0.75视为一致。

**（2）内容抽样审查**：对每个数据集抽取10%的记录样本，进行人工内容审查，评估实际数据内容是否与数据字典一致，是否存在隐蔽的个人信息泄露。

**（3）专家评分**：邀请5位数据安全、信息资源管理、法学领域的专家，对匿名化处理质量进行独立评分。专家评分与编码员评分的差异通过讨论达成一致。

**（4）利用绩效数据收集**：从各平台API获取数据集的下载量、API调用次数、利用案例数等利用绩效指标。对无法通过API获取的平台，采用网页爬虫采集公开统计数据。

### 4.4 数据分析方法

**（1）描述性统计**：对23个平台的匿名化处理质量进行整体描述，计算均值、标准差、分布频率等。

**（2）差异性检验**：采用单因素方差分析（ANOVA）检验东中西部地区平台匿名化质量的差异。

**（3）关联分析**：采用Spearman相关分析和多元回归分析，检验匿名化质量与利用绩效的关联关系。

**（4）倒U型关系检验**：参考Lind和Mehlum（2010）的方法，对匿名化强度与利用绩效的倒U型关系进行统计检验。

| 中部 | 7 | 2.89 | 0.51 |
| 西部/东北 | 6 | 2.56 | 0.48 |

ANOVA结果显示：F(2, 20)=15.32, p<0.001，区域差异显著。事后检验（Tukey HSD）显示东部与中部（p<0.01）、东部与西部（p<0.001）差异显著，中部与西部差异不显著（p=0.12）。

**原因分析**：东部地区平台（如上海、北京、浙江）在技术投入、专业人才、制度规范等方面具有明显优势。上海平台建立了专门的"数据安全审查委员会"，对上架数据集进行逐项匿名化审查；北京平台开发了自动化脱敏工具，可对结构化数据进行批量处理。相比之下，中西部地区平台多依赖人工审查，处理水平参差不齐。

### 5.3 匿名化质量与利用绩效的关联分析

#### 5.3.1 描述性关联

以数据集下载量为利用绩效指标，按AQI等级分组：

| AQI等级 | 数据集数 | 平均下载量 | 中位数下载量 |
|:---|:---|:---|:---|
| 高风险 | 82 | 128 | 85 |
| 中风险 | 216 | 342 | 256 |
| 低风险 | 288 | 1,856 | 1,420 |
| 合规 | 104 | 2,240 | 1,890 |

#### 5.3.2 倒U型关系检验

将AQI作为自变量，下载量（取对数）作为因变量，加入二次项进行回归：

```
ln(Downloads) = β0 + β1·AQI + β2·AQI² + β3·Controls + ε
```

回归结果：

| 变量 | 系数 | 标准误 | t值 | p值 |
|:---|:---|:---|:---|:---|
| AQI | 1.245*** | 0.312 | 3.99 | <0.001 |
| AQI² | -0.186** | 0.072 | -2.58 | 0.010 |
| 数据集主题（控制） | - | - | - | <0.05 |
| 平台经济发展水平（控制） | - | - | - | <0.01 |
| 常数项 | 2.156 | 0.485 | 4.44 | <0.001 |
| R² | 0.328 | | | |
| F统计量 | 28.56*** | | | |

倒U型关系显著。通过求导计算拐点：AQI* = -β1/(2·β2) = -1.245/(2×(-0.186)) ≈ 3.35。

**核心发现**：当AQI<3.35时，匿名化质量提升促进下载量增长；当AQI>3.35时，过度匿名化导致数据可用性下降，反而抑制下载量。当前23个平台的AQI均值为3.12，整体位于倒U型曲线的上升段，说明**大多数平台尚未达到"过度匿名化"的临界点，仍有提升空间**。

#### 5.3.3 机制分析

为解释倒U型关系的机制，本研究从"利用者信任"和"数据可用性"两个中介变量进行分析：

**（1）利用者信任机制**：匿名化质量较低的平台，数据集可能包含可识别的个人信息，利用者担心法律风险，下载意愿降低。问卷调查显示，68%的数据利用者表示"个人信息泄露风险"是其选择数据集时考虑的重要因素。

**（2）数据可用性机制**：过度匿名化（如对所有数值字段进行区间化处理）可能破坏数据的统计分布特征，影响分析结果的有效性。访谈发现，42%的研究者曾因"匿名化过度导致数据无法分析"而放弃使用某数据集。

### 5.4 典型问题识别

通过对高风险和中风险数据集的深度审查，识别出以下典型问题：

**问题1：直接标识符残留（占问题数据集的38%）**
部分数据集名义上去除了"姓名"字段，但在"备注""描述"等字段中保留了完整的个人姓名。例如，某平台的"低保人员名单"数据集中，"备注"字段包含"张三，身份证号XXX，家庭住址XXX"等完整个人信息。

**问题2：准标识符组合识别（占问题数据集的31%）**
单独看每个字段都不具有识别性，但字段组合后可唯一识别个人。例如，某平台的"学生成绩"数据集包含"学校+年级+班级+性别+出生年月"五个字段，在小型学校中，该组合可唯一识别90%以上的学生。

**问题3：过度匿名化（占问题数据集的21%）**
部分平台为规避法律风险，采取了"一刀切"的过度匿名化处理。例如，将所有地理位置信息精确到省级，将年龄信息模糊为"青年/中年/老年"区间，导致数据集丧失地理分析和年龄细分的能力。

**问题4：匿名化处理不一致（占问题数据集的10%）**
同一平台的不同数据集采用不同的匿名化处理标准。例如，"人口统计"数据集对年龄进行了区间化处理，而"医疗健康"数据集却保留了精确年龄，处理标准的混乱增加了利用者的认知成本。

## 六、分级分类处理策略

### 6.1 "数据集敏感度×匿名化强度"匹配矩阵

基于评估结果，本研究提出"数据集敏感度×匿名化强度"匹配矩阵，为平台提供差异化的匿名化处理策略：

| 数据集类型 | 敏感度 | 推荐匿名化强度 | 处理策略 | 目标AQI |
|:---|:---|:---|:---|:---|
| 人口统计 | 高 | 高强度 | k-匿名(k≥5) + 差分隐私(ε≤1) | 4.2-4.8 |
| 医疗健康 | 高 | 高强度 | k-匿名(k≥5) + 敏感属性泛化 | 4.2-4.8 |
| 交通轨迹 | 高 | 高强度 | 空间模糊化(≥500m) + 时间区间化(≥1h) | 4.0-4.5 |
| 环境监测 | 中 | 中强度 | 空间模糊化(≥100m) | 3.5-4.0 |
| 经济数据 | 中 | 中强度 | 数值区间化(≥10%) | 3.2-3.8 |
| 教育资源 | 低 | 低强度 | 直接标识符移除 | 3.0-3.5 |
| 文化旅游 | 低 | 低强度 | 直接标识符移除 | 3.0-3.5 |
| 地理信息 | 中 | 中强度 | 空间模糊化(≥50m) | 3.5-4.0 |

### 6.2 平台分级治理建议

根据当前匿名化处理质量，对23个平台提出分级治理建议：

**A级平台（AQI≥4.0，上海、北京）**：
- 继续保持高标准，可作为全国示范平台
- 探索自动化匿名化工具的研发与应用
- 参与国家数据局匿名化处理评价标准的试点验证

**B级平台（3.5≤AQI<4.0，浙江、广东、江苏等6个）**：
- 重点解决准标识符组合识别问题
- 引入外部安全审计机制
- 建立数据集匿名化处理SOP（标准作业程序）

**C级平台（3.0≤AQI<3.5，湖北、四川、福建等8个）**：
- 建立专门的数据安全审查团队
- 引入自动化脱敏工具
- 开展平台运营人员的匿名化处理培训

**D级平台（AQI<3.0，山西、内蒙古、江西等7个）**：
- 优先解决直接标识符残留问题（最基础、最紧迫）
- 暂停上架高敏感度数据集，待整改完成后恢复
- 接受省级数据管理部门的专项督导

## 七、讨论

### 7.1 理论贡献

**第一**，本研究将"数据流通匿名化处理评价标准"从制度文本转化为可操作的评估工具，填补了标准制定与平台实践之间的评估空白。评估指标体系可直接用于标准的场景化应用和效果验证。

**第二**，研究发现的匿名化质量与利用绩效的倒U型关系，丰富了政府数据开放评价的理论框架。现有研究多假设"匿名化越强越好"或"开放越多越好"，本研究揭示了两者之间的非线性权衡关系。

**第三**，研究提出的"数据集敏感度×匿名化强度"匹配矩阵，为差异化的分级分类处理提供了可操作的工具，超越了现有"一刀切"处理模式的局限。

### 7.2 政策启示

**第一**，对国家数据局的启示：正在制定的"数据流通匿名化处理评价标准"应充分考虑政府数据开放平台的特殊性，在通用标准基础上增加"平台场景化条款"。

**第二**，对省级平台管理者的启示：匿名化处理不应被视为单纯的合规负担，而应作为提升数据利用绩效的战略工具。适度投资匿名化处理能力建设，可带来数据利用率的显著提升。

**第三**，对数据利用者的启示：应建立"匿名化质量认证"机制，为高质量匿名化数据集提供标识，降低利用者的信息筛选成本。

### 7.3 研究局限与未来方向

**第一**，样本局限性。本研究仅评估了690个数据集，占23个平台总数据集的极小部分。未来研究可扩大样本量，甚至实现全量评估。

**第二**，方法局限性。本研究主要采用人工审查和专家评分，未来可引入自动化匿名化质量检测工具，提升评估效率和客观性。

**第三**，纵向追踪不足。本研究为横截面评估，未来可开展纵向追踪研究，考察平台匿名化处理水平的动态变化。

---

## 八、结论与建议

### 8.1 核心结论

（1）当前我国省级政府数据开放平台的匿名化处理质量整体处于中等水平（AQI均值3.12），仅15.1%的数据集达到合规标准，43.2%存在不同程度的风险。

（2）匿名化质量与数据利用绩效之间存在显著的倒U型关系，拐点位于AQI≈3.35。当前多数平台位于曲线的上升段，仍有提升空间。

（3）区域差异显著，东部省份平台显著优于中西部平台，技术投入和制度规范是主要影响因素。

（4）平台匿名化处理存在"直接标识符残留""准标识符组合识别""过度匿名化""处理不一致"四类典型问题。

### 8.2 对策建议

**（1）建立全国统一的政府数据开放平台匿名化处理规范**
建议国家数据局在制定"数据流通匿名化处理评价标准"时，增设"政府数据开放场景化附录"，明确不同主题数据集的匿名化处理要求和达标标准。

**（2）实施平台匿名化处理能力分级认证**
借鉴信息安全等级保护制度，建立平台匿名化处理能力的A/B/C/D四级认证体系，将认证结果与平台考核、资金支持挂钩。

**（3）推广"数据集敏感度×匿名化强度"匹配矩阵**
建议各平台根据本研究提出的匹配矩阵，对现有数据集进行重新审查和分类处理，实现差异化、精细化的匿名化管理。

**（4）加强中西部地区平台的技术帮扶**
建议东部地区的A级平台与中西部D级平台建立"结对帮扶"机制，通过技术输出、人员培训、经验共享等方式提升整体水平。
