# Kimi任务指令：全文脚注引用格式转换

## 任务概述

将V20论文（`docs/博士论文_最终定稿版_v10.md`）中的行内引用统一转换为GB/T 7714-2015格式的脚注引用。

## 格式规范（基于武汉大学博士论文模板PDF分析）

### 脚注编号
- Markdown中使用 `[^1]`, `[^2]`, `[^3]` ... 格式
- 在文档末尾集中列出所有脚注内容
- pandoc转换为Word时会自动处理为页面底部脚注

### 脚注内容格式

**中文期刊论文：**
```
[^N]: 作者.题名[J].刊名,年,卷(期):页码.
```
例：`[^1]: 田宏. 全球研究理事会（GRC）2014年全体大会在北京召开[J]. 中国科技期刊研究, 2014, 25(6):862.`

**英文期刊论文：**
```
[^N]: 作者.题名[J].刊名,年,卷(期):页码.
```
例：`[^2]: Nature Methods Editorial Board. Reviewing computational methods[J]. Nature Methods, 2015, 12(12):1099.`

**学位论文：**
```
[^N]: 作者.题名[D].学校,年.
```
例：`[^3]: 常静. 百度百科及开源社区参与者的动机与行为关系的实证研究[D]. 华南理工大学, 2010.`

**专著：**
```
[^N]: 作者.书名[M].出版地:出版社,年.
```

**网络文献：**
```
[^N]: 作者.题名[EB/OL].(发布日期)[引用日期].URL.
```

**多作者（4人及以上）：**
```
[^N]: 作者1, 作者2, 作者3, et al.题名[J].刊名,年,卷(期):页码.
```

## 转换规则

### 1. 现有引用模式识别

V19中存在以下引用模式需要转换：

**模式A：方括号数字引用（如[1], [2]）**
- 例：`OECD[2]报告`、`Data.gov[3]`、`Goodhart法则[4]`、`Mark Moore[5]`
- 转换：将 `[N]` 改为 `[^N]`，并在文档末尾添加脚注内容

**模式B：行内作者-年份引用**
- 例：`郑磊等学者`、`张会平进一步区分`、`Rawls在《正义论》中提出`、`Norris所指出的`
- 转换：在作者或观点后添加 `[^N]`，并在文档末尾添加脚注内容

**模式C：政策文件引用**
- 例：`《促进大数据发展行动纲要》`、`《数据安全法》`、`"数据二十条"`
- 转换：在政策名称后添加 `[^N]`，并在文档末尾添加脚注内容

**模式D：数据/事实引用**
- 例：`复旦大学数字与移动治理实验室发布的《中国开放数林指数[1]》报告显示`
- 转换：将 `[1]` 改为 `[^1]`，并在文档末尾添加脚注内容

### 2. 脚注编号规则

- 全文统一编号，从 `[^1]` 开始连续编号
- 同一文献多次引用时，每次引用都使用新的脚注编号（不能复用）
- 脚注内容可以重复（同一文献在不同位置引用时，脚注内容相同但编号不同）

### 4. 参考文献列表

在论文末尾的"参考文献"章节，使用顺序编码制：
```
[1] 作者.题名[J].刊名,年,卷(期):页码.
[2] 作者.题名[D].学校,年.
```

参考文献列表中的文献应与脚注中的文献一一对应，但：
- 参考文献列表：按编号顺序排列，不标注具体页码
- 脚注：在引用位置标注，可标注具体页码

## 操作步骤

1. **通读全文**，识别所有需要添加脚注的引用点
2. **建立文献库**：为每个引用点分配脚注编号，记录文献信息
3. **批量替换**：使用文本编辑器的查找替换功能，将 `[N]` 替换为 `[^N]`
4. **添加脚注内容**：在文档末尾添加所有脚注内容
5. **添加参考文献列表**：在论文末尾添加"参考文献"章节
6. **检查一致性**：确保脚注编号连续、无遗漏、无重复

## 注意事项

1. 所有新增文献必须真实可查，不得编造
2. 英文文献作者姓名格式统一（姓在前或按原文）
3. 期刊名、书名等使用原文（英文用英文，中文用中文）
4. 年份后不加"年"字（但中文文献有时保留，需统一）
5. 页码用冒号分隔，不加"页"字
6. 多作者时，3人以内全部列出，4人及以上用"et al."或"等"

## 输出要求

1. 修改后的Markdown文件：`docs/博士论文_最终定稿版_v20.md`
2. 脚注内容完整、格式统一
3. 参考文献列表在论文末尾，数量150条以上，外文30%以上
