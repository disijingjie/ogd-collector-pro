## 方案C：内网穿透（最快，今晚就能用）

### 原理
你本地Flask已经在跑（127.0.0.1:5000），通过内网穿透工具把本地端口映射到公网URL，老师直接访问公网链接即可。

### 步骤（Windows）

#### 1. 下载 cpolar
- 访问 https://www.cpolar.com/download
- 下载 Windows 版本，安装

#### 2. 注册账号
- 访问 https://dashboard.cpolar.com/signup
- 用邮箱注册，获取 Authtoken

#### 3. 配置并启动
```powershell
# 打开PowerShell，进入cpolar安装目录
cd "C:\Program Files\cpolar"

# 认证（把xxx换成你的token）
.\cpolar.exe authtoken xxxxxxxxxxxxxxxxxxxxxxx

# 映射本地5000端口到公网（http）
.\cpolar.exe http 5000
```

#### 4. 获取公网URL
运行后会显示类似：
```
Forwarding  http://abc123.cpolar.top -> http://localhost:5000
Forwarding  https://abc123.cpolar.top -> http://localhost:5000
```

把 `http://abc123.cpolar.top` 发给老师即可访问你的系统。

**注意**：免费版URL每次重启会变化。如需固定URL，需付费（约6元/月）。

## 方案B：Render云部署（免费，长期稳定）

### 原理
Render是一个海外PaaS平台，支持免费部署Python Flask应用，提供 `xxx.onrender.com` 子域名，无需备案。

### 优点
- 完全免费
- 无需域名、无需备案
- 代码推送到GitHub后自动部署
- 适合给老师长期展示

### 缺点
- 国内访问可能略慢（海外服务器）
- 免费实例30分钟无访问会休眠，首次访问需等待10秒唤醒

### 步骤

#### 1. 创建 requirements.txt（已有）
确认 `ogd_collector_system/requirements.txt` 包含：
```
Flask>=2.3.0
requests>=2.31.0
beautifulsoup4>=4.12.0
pandas>=2.0.0
numpy>=1.24.0
lxml>=4.9.0
gunicorn>=21.0.0
```

#### 2. 创建入口文件 `wsgi.py`
在 `ogd_collector_system/` 目录下新建 `wsgi.py`：
```python
from app import app

if __name__ == "__main__":
    app.run()
```

#### 3. 上传到GitHub
```bash
cd ogd_collector_system
git init
git add .
git commit -m "Initial commit"
# 在GitHub新建仓库，然后推送
git remote add origin https://github.com/你的用户名/ogd-collector.git
git push -u origin main
```

#### 4. 在Render部署
1. 访问 https://dashboard.render.com/
2. 用GitHub账号登录
3. 点击 "New +" → "Web Service"
4. 选择你的GitHub仓库
5. 配置：
   - Name: `ogd-collector`
   - Runtime: `Python 3`
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `gunicorn -b 0.0.0.0:10000 wsgi:app`
6. 点击 "Create Web Service"

等待2-3分钟，系统会自动部署完成，给你分配一个类似 `https://ogd-collector.onrender.com` 的URL。

## 方案A：腾讯云轻量服务器（正式长期部署）

### 费用预算
| 项目 | 费用 | 说明 |
|------|------|------|
| 腾讯云轻量服务器 | ~40-60元/年 | 新用户首年优惠，2核2G配置 |
| 域名（.cn） | ~30元/年 | 腾讯云/阿里云购买 |
| ICP备案 | 免费 | 必须，约7-20天 |
| **合计** | **~70-90元/年** | |

### 部署流程

#### 第一阶段：购买服务器（1天）
1. 访问 https://cloud.tencent.com/product/lighthouse
2. 选择 "通用型"，2核2G，3M带宽
3. 选择 **Ubuntu 22.04** 系统镜像
4. 购买时长1年（新用户约50元）
5. 记住公网IP和root密码

#### 第二阶段：购买域名（1天）
1. 访问 https://dnspod.cloud.tencent.com/
2. 搜索并购买域名（如 `wmlab.cn` 或 `wenmingdata.cn`）
3. 约30-40元/年

#### 第三阶段：域名备案（7-20天）
1. 在腾讯云备案系统提交资料
2. 需要：身份证照片、人脸识别、域名证书
3. 腾讯云初审1-3天 → 管局审核7-20天
4. **备案期间网站不能访问**

#### 第四阶段：部署代码（1天）
```bash
# 1. SSH登录服务器
ssh root@你的IP

# 2. 安装Python和依赖
apt update
apt install -y python3 python3-pip nginx git
pip3 install flask requests beautifulsoup4 pandas numpy lxml gunicorn

# 3. 上传代码（本地执行）
scp -r ogd_collector_system root@你的IP:/opt/

# 4. 在服务器上初始化数据库
cd /opt/ogd_collector_system
python3 -c "from models import init_db, init_platforms_data; init_db(); init_platforms_data()"

# 5. 配置systemd服务
cat > /etc/systemd/system/ogd.service << 'EOF'
[Unit]
Description=OGD Collector Pro
After=network.target

[Service]
User=root
WorkingDirectory=/opt/ogd_collector_system
ExecStart=/usr/bin/gunicorn -b 127.0.0.1:5000 --workers 2 wsgi:app
Restart=always

[Install]
WantedBy=multi-user.target
EOF

systemctl enable ogd
systemctl start ogd

# 6. 配置Nginx反向代理
cat > /etc/nginx/sites-available/ogd << 'EOF'
server {
    listen 80;
    server_name 你的域名;

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
EOF

ln -s /etc/nginx/sites-available/ogd /etc/nginx/sites-enabled/
nginx -t
systemctl restart nginx
```

## 方案选择建议

### 如果今晚就要给老师看
→ **方案C（cpolar）**，5分钟搞定，URL发给老师即可。

### 如果想长期给老师/答辩委员会展示
→ **方案B（Render）**，免费、无需备案、有固定URL。

### 如果追求国内访问速度+正式感（比如挂在简历/学术主页上）
→ **方案A（腾讯云）**，备案后最正规，约1-2周完成。
