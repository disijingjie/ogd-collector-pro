# 第八章 结论与展望

## 8.1 主要结论

本研究以"政府数据开放平台数据利用绩效如何评估、差异为何产生、如何改进"为核心逻辑主线，在批判性反思现有"供给导向"评估范式局限性的基础上，将公共价值理论、数据要素价值化理论与组态理论进行整合，构建了"评估→效率→归因→组态"的多方法递进分析框架。通过对我国28个建有独立数据开放平台的省级行政区开展实证研究，得出以下四项核心结论。

**结论一：构建了面向受众利用与价值转化的4E全链条评估框架，形成评估范式从"供给导向"向"效果导向"的转型。**

本研究基于4E框架（供给保障-平台服务-数据质量-利用效果-公平性），构建了涵盖39项三级指标的评估体系，将"利用效果"从辅助维度提升为核心维度（权重0.30），形成从"供给侧评估"到"效果导向评估"的范式转变。实证结果显示，28个平台的利用效果维度得分均值仅0.31，远低于供给保障（0.52）、平台服务（0.48）等维度，表明"数据开放了但没人用"是全国性的普遍困境，效果导向评估势在必行。

**结论二：揭示了"数据口径幻觉"现象的普遍性及其对评估结果的扭曲效应，提出了"口径一致性系数"纠偏工具。**

本研究发现，28个平台的数据目录数量与有效数据集数量之间存在巨大差异，口径一致性系数中位数仅为0.122——平均而言，目录中约8个数据集仅有1个是真正有效的。这一现象揭示了"数量导向"评估范式对平台行为的扭曲引导——平台运营者有动力批量上传空壳数据集以最大化目录数量。本研究将"口径一致性系数"操作化为可测量的纠偏指标，为纠正指标扭曲的行为偏差提供了量化工具。

**结论三：识别了数据质量作为绩效差异的"传导中枢"，揭示了高绩效的三条多重等效路径。**

DEMATEL分析显示，数据质量的中心度最高（2.85），处于原因因素和结果因素的交界地带，是影响绩效的传导中枢。fsQCA分析揭示了三条高绩效路径：H1技术驱动型（高供给×高服务×高质量）、H2效率-区域型（高质量×高技术×高区域经济）、H3生态协同型（高供给×高利用×高技术）。三条路径的一致性均超过0.85，总覆盖度为0.68，表明高绩效存在多重等效的实现方式。同时，DEMATEL分析显示制度环境是最强的原因因素（原因度0.72），揭示了制度完善是绩效提升的"治本之策"。

**结论四：发现了东中西部之间显著的"效果鸿沟"，为分类施策和区域协同发展提供了实证依据。**

区域差异分析显示，东部利用效果均值0.58，中部0.28，西部0.24——东部是中部的2.1倍，是西部的2.4倍。更为关键的是，中西部平台的问题不是"数据不够多"（供给保障维度差距相对较小），而是"数据没人用"（利用效果维度差距极为悬殊）。这一发现为差异化对策设计提供了方向——中西部平台的优化策略应从"增加数据开放"转向"培育利用生态"。

## 8.2 理论贡献

本研究的理论贡献主要体现在以下四个方面。

**第一，推动了政府数据开放评估范式从"供给导向"向"效果导向"的转型。** 本研究将4E框架从公共财政领域迁移到政府数据开放领域，构建了涵盖"供给-平台-质量-效果-公平性"的全链条评估框架。这一框架将绩效的界定从"开放了多少"拓展为"被利用了多少"，从"过程指标"转向"结果指标"，回应了公共价值理论对绩效本质的追问——公共资源的绩效不应仅以"是否可获取"为标准，更应以"是否产生了实际社会价值"为评判依据。这一范式转变对于公共部门绩效评估理论具有普遍性的启示意义。

**第二，提出了"数据口径幻觉"概念，丰富了数据开放评估的理论话语。** "数据口径幻觉"概念揭示了评估指标对行为激励的深层影响——当评估指标与真实绩效脱节时，被评估对象的行为策略就会发生系统性扭曲。这一概念对于公共部门绩效评估具有普遍性的方法论意义：在任何以量化指标为基础的评估体系中，都存在着"指标失真"的风险，评估设计者需要不断审视指标与真实绩效之间的对应关系，防止"指标目标化"导致的系统性偏差。

**第三，采用多方法三角验证，为政府数据开放评估提供了方法论示范。** 本研究综合运用TOPSIS、DEA、DEMATEL、fsQCA四种方法，从不同角度分析同一问题，形成了多方法三角验证。四种方法的结果相互印证、相互补充，增强了研究结论的稳健性和可信度。这种多方法集成设计超越了现有研究单一方法的局限，为复杂公共管理现象的评估提供了方法论参考。

**第四，基于组态分析设计了差异化对策，超越了"一刀切"的政策建议。** 本研究基于fsQCA识别的三条高绩效路径，设计了"技术驱动型""效率-区域型""生态协同型"三种差异化优化策略。这种"组态-对策"对应关系使政策建议更具针对性和可操作性，为分类施策提供了实证依据。这一研究思路对于公共政策领域的"因地制宜"原则提供了方法论支撑。

## 8.3 实践启示

本研究的实践启示主要体现在以下四个方面。

**第一，评估体系应引入效果导向指标，引导平台行为回归"有效供给"本质。** 建议在现有的评估体系中增加利用效果维度的权重，引入"有效数据集数量""口径一致性系数""应用成果数量"等效果导向指标，降低"数据目录数量"等数量导向指标的权重。通过评估体系的"指挥棒"作用，引导平台运营者从"追求开放数量"转向"追求利用效果"。

**第二，数据质量是绩效提升的"牛鼻子"，应建立三重质量保障机制。** 建议建立"上传前审查+上传后抽检+用户反馈纠错"的三重质量保障机制，制定统一的数据质量标准和元数据规范，推动各平台在数据格式、字段命名、编码规则等方面的标准化。数据质量的提升是改善利用效果的最直接、最有效的途径。

**第三，制度完善是绩效提升的"治本之策"，应加快数据开放法规政策体系建设。** 建议在国家层面出台《政府数据开放条例》，在地方层面完善数据开放管理办法，建立数据治理机构，制定数据质量标准，建立绩效考核机制。制度环境的完善是技术优化、服务改进、质量提升的基础保障。

**第四，分类施策是绩效优化的有效路径，应建立平台分类管理制度。** 建议根据平台的绩效水平和资源禀赋，将平台分为领先型、追赶型、起步型三类，制定差异化的发展目标、支持政策和考核标准。同时，建立东中西部平台之间的结对帮扶机制，推动区域间的数据标准统一、平台互联互通、应用成果共享。

## 8.4 研究局限

本研究存在以下局限，需要在后续研究中加以改进。

**第一，样本量有限。** 本研究仅包含28个省级平台，样本量相对较小，可能影响部分统计方法的稳健性（如fsQCA建议样本量大于30）。虽然28个样本覆盖了我国绝大多数建有独立平台的省级行政区，但研究结论的推广范围仍需谨慎对待。特别是，副省级和地市级平台未被纳入样本，这些平台的绩效特征可能与省级平台存在差异。

**第二，截面数据限制。** 本研究采用截面数据（2025年1-3月的数据），无法分析平台绩效的动态变化。开放数林指数提供了2017-2025年的时间序列数据，但本研究未充分利用这些数据进行纵向比较和趋势分析。后续研究可以构建面板数据模型，分析平台绩效的演化规律和影响因素的时间效应。

**第三，数据可得性约束。** 部分关键指标（如下载量、API调用量、用户访问量）多数平台未公开，本研究采用爬虫采集和插补方法获取这些数据，可能存在一定的测量误差。虽然本研究采取了数据质量控制措施，但数据可得性的约束仍对评估结果的精确性产生影响。

**第四，方法局限性。** TOPSIS方法对指标权重敏感，权重设定的偏差会直接影响排序结果；DEA方法对投入产出指标的选择敏感，不同的指标组合可能产生不同的效率结果；fsQCA方法的样本量临界问题（28个样本处于临界状态）可能影响结果的稳健性；DEMATEL方法依赖于专家评分，专家的主观判断可能影响因果关系的识别。虽然本研究通过多方法三角验证增强了结论的稳健性，但单一方法的局限性仍需在研究结论的解释中予以说明。

**第五，缺乏用户视角的需求侧评估。** 本研究的评估主要基于平台供给数据和客观指标，缺乏对数据用户（如企业、开发者、研究人员）的需求和体验的直接测量。用户满意度、需求匹配度、使用障碍等需求侧指标未被纳入评估体系，可能导致评估结果与用户真实体验之间存在偏差。

## 8.5 未来展望

基于上述研究局限，本研究提出以下未来研究方向。

**第一，扩大样本范围，开展多层次比较研究。** 将副省级和地市级平台纳入样本，构建"省级-副省级-地市级"的多层次比较研究框架，分析不同层级平台在绩效水平、影响因素、优化路径方面的差异。同时，开展国际比较研究，将我国平台与欧美日韩等国家的平台进行对标分析，识别我国平台的优势和短板。

**第二，构建面板数据，开展动态演化分析。** 利用开放数林指数2017-2025年的时间序列数据，结合本研究采集的截面数据，构建面板数据集，运用动态面板模型、事件研究法等方法，分析平台绩效的演化规律、政策冲击效应和追赶路径。特别是，可以分析2015年《促进大数据发展行动纲要》、2020年数据要素化政策、2023年国家数据局成立等重大政策事件对平台绩效的影响。

**第三，引入用户视角，开展需求侧评估。** 通过问卷调查、深度访谈、用户行为分析等方法，直接从数据用户的角度评估平台绩效，测量用户满意度、需求匹配度、使用障碍、改进建议等需求侧指标。将需求侧评估与供给侧评估相结合，形成更为全面的评估视角。

**第四，探索AI辅助评估方法。** 随着人工智能技术的发展，可以探索利用大语言模型（LLM）、机器学习等方法辅助数据开放评估。例如，利用LLM自动提取和解析数据集的元数据信息，利用机器学习模型预测数据集的潜在利用价值，利用自然语言处理技术分析用户反馈中的情感倾向和改进建议。AI辅助评估有望提高评估的效率、客观性和预测能力。

**第五，深化授权运营评估研究。** 随着公共数据授权运营从"试点"走向"规范"，授权运营在数据利用中的比重将持续提升。后续研究可以专门设计授权运营的评估指标体系，分析授权运营对平台整体绩效的贡献，探索授权运营与直接开放的协同机制，为授权运营的规范化发展提供评估工具和理论支撑。

**第六，开展无平台省份的反向案例研究。** 青海、甘肃、西藏三个省份尚未建立独立的数据开放平台，其"无平台"状态本身就是一个值得深入研究的现象。后续研究可以采用案例研究方法，分析这三个省份未建平台的原因（是经济条件限制、制度意愿不足、还是数据资源匮乏？），为其他潜在的"无平台"地区提供参考，也为全国层面的平台布局优化提供决策依据。
