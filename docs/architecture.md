# OGD-Collector Pro 系统架构说明

## 一、系统概述

**OGD-Collector Pro**（Open Government Data Collector Professional）是专为博士论文研究设计的三层架构政府数据开放平台采集与可视化系统。系统支持对中国大陆31个省级、13个副省级/计划单列市以及287个地级市（分层抽样60个）的数据开放平台进行自动化采集、评估与可视化分析。

## 三、核心功能模块

### 3.1 采集引擎 (CollectorEngine)

```python
class CollectorEngine:
    - detect_platform(url): 检测平台可用性及功能特征
    - calculate_4e_scores(details): 基于4E框架计算综合评分
    - collect_single(platform): 采集单个平台
    - run_collection(filter): 运行完整采集流程
    - pause/resume/stop: 采集控制
```

**采集指标（4E框架）**：

| 维度 | 代码 | 指标 | 权重 |
|------|------|------|------|
| 供给保障 | C1 | 数据集数量、格式多样性、API、批量下载 | 25% |
| 平台服务 | C2 | 响应时间、HTTPS、搜索、下载、可视化 | 25% |
| 数据质量 | C3 | 更新信息、元数据、反馈渠道 | 25% |
| 利用效果 | C4 | 注册、预览、应用成果 | 25% |

### 3.2 实时监控系统

- **SSE (Server-Sent Events)** 实时推送采集进度
- 每10个平台自动保存中间状态快照
- 支持断点续采和异常恢复

### 3.3 可视化分析

- **层级对比**: 雷达图、绩效梯度柱状图、可用率对比
- **区域分析**: 区域分布热力图、东中西部对比
- **4E维度**: 箱线图、相关性矩阵、功能支持率
- **TOP排行**: 综合得分排名、维度分解展示

## 四、数据库设计

### 4.1 实体关系图

```
platforms (1) ───< (N) collection_records
      │
      └──< (N) collection_tasks
      │
      └──< (N) collection_snapshots
      │
      └──< (N) analysis_results
```

### 4.2 核心表结构

**platforms**: 平台基础信息（104条预置数据）
- code, name, tier, region, province, type, url, url_pattern

**collection_tasks**: 采集任务
- task_name, task_type, status, total_count, completed_count, success_count, fail_count

**collection_records**: 采集记录（核心数据）
- 平台基础信息 + 4E维度指标 + 功能检测位 + 综合评分

**collection_snapshots**: 中间状态快照
- 支持断点续采和采集过程回溯

## 六、部署与使用

### 6.1 环境要求
- Python 3.8+
- 无需额外服务器（SQLite内置）
- 无需外部数据库

### 6.2 启动步骤
```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 启动系统
python app.py

# 3. 访问Web界面
open http://127.0.0.1:5000
```

### 6.3 采集流程
1. 进入"采集管理"页面
2. 点击"新建采集任务"
3. 选择采集类型（完整/省级/副省级/地级市）
4. 点击"开始采集"
5. 实时监控采集进度和日志
6. 采集完成后查看"数据看板"和"分析图表"
