# 博士学位论文评审意见

**论文题目：** 中国省级政府数据开放平台数据利用绩效评估与优化路径研究——基于4E框架的实证分析

**评审日期：** 2026年4月27日

---

## 一、论文核心内容概括

本研究聚焦政府数据开放领域"重供给轻效果"的评估偏差，构建基于4E框架（供给保障-平台服务-数据质量-利用效果-公平性）的绩效评估体系，涵盖5个一级维度、18个具体指标。以自主开发的OGD-Collector Pro系统采集全国23个省级平台数据，综合运用TOPSIS、DEA-BCC、DEMATEL、fsQCA和多期DID方法，系统评估平台绩效、识别效率损失来源、揭示因素因果网络与组态路径，并评估"数据二十条"政策效应。核心发现：山东等头部平台绩效显著领先；DEA有效平台仅1个；E4利用效果为fsQCA必要条件；政策效应存在维度差异和时滞特征。

## 二、分维度逐项评审

### （一）研究意义与价值

**优点：**
- 选题契合国家数字政府建设和数据要素市场化改革战略需求，具有明确的政策相关性；
- 针对"数据口径幻觉"现象提出原创性概念，揭示了现有评估体系"重数量轻质量"的结构性偏差，问题意识敏锐；
- 4E框架将"利用效果"提升为核心维度，回应了公共价值理论对绩效本质的追问，理论定位清晰。

**不足：**
- 对"为什么要评估省级平台而非地市级平台"的论证不足。我国地市级平台数量超过200个，与公众距离更近，研究对象的选择缺乏充分的学理论证（第1章1.3.1节）；
- "数据口径幻觉"概念虽具原创性，但其理论渊源与信息经济学中"信息不对称"、公共管理学中"指标博弈"（gaming the metrics）等既有概念的区分度未充分阐明，存在"新瓶装旧酒"的风险（第1章1.1.3节）；
- 研究意义部分对国际学术对话的贡献阐述较弱，未能清晰说明本研究如何回应国际OGD评估研究的前沿议题。

**修改建议：**
1. 在第1章1.3.1节补充样本选择论证，明确说明"省级平台作为研究对象"的理论依据——可从行政层级代表性、数据完整性、可比性三个维度论证；
2. 在第1章1.1.3节增加"数据口径幻觉"与既有概念（信息不对称、指标博弈、数字虚荣）的辨析，明确概念边界和理论增量；
3. 在1.2节补充国际学术意义，说明本研究对国际OGD评估研究的贡献（如首个系统性引入fsQCA的OGD研究）。

### （二）创新性

**优点：**
- 方法组合创新显著：首次将TOPSIS、DEA、DEMATEL、fsQCA、DID五种方法整合应用于政府数据开放评估，形成"评估-归因-组态-因果"的完整方法链；
- 提出"口径一致性系数"并建立五类口径转换系数体系（1.0/0.8/0.5/0.3/0.4），具有可操作的方法论贡献；
- 自主开发OGD-Collector Pro采集系统，实现了多平台并发自动化采集，数据采集的独立性和客观性优于依赖自报数据的国际评估工具。

**不足：**
- **创新点表述存在夸大倾向。** 摘要将fsQCA分析结果表述为"三条核心组态路径"（覆盖率31.2%、28.7%、19.8%），但英文摘要和第六章正文（6.3节）表述为"两条核心路径"（覆盖率81%和19%），前后矛盾；
- **理论创新深度有限。** 4E框架源于公共财政评估领域，本研究将其"移植"到数据开放领域，但框架的理论重构论证不够充分——四个维度在数据开放场景下的理论内涵与传统公共财政场景的差异未系统展开；
- **"诊断科学"范式的提法过于宏大。** 第2章2.6.3节提出从"评估科学"到"诊断科学"的范式转型，但"诊断科学"并非公认的学术概念，作为博士论文的原创性理论贡献，其学术合法性需要更充分的论证。

**修改建议：**
1. **立即修正数据矛盾：** 统一fsQCA路径数量（2条或3条）及覆盖率数据，全文交叉核对；建议以第六章正文为准（2条路径：H1覆盖率81%，H2覆盖率19%），修正摘要和第七章的表述；
2. 在第3章3.3节增加"4E框架的语境化修正"小节，系统论证四个维度在政府数据开放场景下的理论内涵重构；
3. 将"诊断科学"改为"诊断式评估"（diagnostic evaluation），定位为方法论取向而非"范式转型"，降低理论 claims 的强度。

### （三）研究方法

**优点：**
- 多方法三角验证设计科学：TOPSIS与DEA在绩效评估环节交叉验证，DEMATEL与fsQCA在影响因素分析环节交叉验证，增强了结论稳健性；
- 数据采集方法规范：自动化爬虫+人工核验的双轨机制，数据口径标准化处理（五类口径转换系数），确保了数据质量；
- 政策评估方法选择恰当：多期DID方法用于评估"数据二十条"政策效应，符合政策评估的因果推断要求。

**不足：**
- **DEA方法的投入-产出指标设计存在逻辑问题。** 第4章4.3.3节将"数据集数量""API接口数"等同时作为投入和产出指标，但DEA要求投入和产出必须是不同的变量集。同一指标既作投入又作产出，会导致效率测度失真；
- **DEMATEL分析缺乏专家权重基础。** 第6章6.2节DEMATEL分析直接基于平台得分计算因素关联矩阵，但未说明专家数量和背景。标准的DEMATEL方法需要多位专家独立打分后取均值，本研究的操作方式偏离了方法规范；
- **fsQCA校准阈值设定缺乏敏感性分析的充分呈现。** 第6章6.3节提及进行了校准阈值调整，但未报告不同阈值下的分析结果对比，稳健性检验的透明度不足；
- **"数据二十条"的DID设计存在内生性隐患。** 政策实施时间为2022年12月，但数据采集时间为2025年3月，处理组和对照组的划分依赖于平台是否在政策后发生显著变化，这一划分标准可能存在主观性。

**修改建议：**
1. **修正DEA指标设计：** 重新界定投入指标（建议：平台运营年限、财政投入、人员编制）和产出指标（数据集数量、API调用量、应用成果数量、用户访问量），确保投入-产出变量不重叠；
2. **补充DEMATEL专家信息：** 在第六章增加专家数量、专业背景、独立打分-聚合过程的说明；若未进行专家打分，应明确说明矩阵计算的替代方法及其合理性；
3. **补充fsQCA敏感性分析：** 在第六章6.3.6节增加校准阈值（0.75/0.80/0.85）和一致性阈值（0.75/0.80/0.85）组合的完整分析结果表格；
4. **补充DID稳健性检验：** 增加平行趋势检验图、安慰剂检验结果，以及不同时间窗口下的效应估计。

## 三、总体评价

### 论文水平判断

**建议：修改后再审。**
