# OGD-Collector Pro：三层架构数据开放平台采集与可视化系统

## 一、系统定位与研究价值

OGD-Collector Pro（Open Government Data Collector Professional）是本研究自主设计开发的**三层架构政府数据开放平台采集与评估系统**。该系统不仅是本论文实证研究的数据采集工具，更是论文方法论创新的核心载体。系统实现了从"平台检测到4E评估"的完整闭环，将理论框架（4E评估框架）直接嵌入技术实现，形成了"理论-技术-数据"三位一体的研究基础设施。

**系统访问地址**: http://127.0.0.1:5000（本地部署）
**系统版本**: v2.0
**开发日期**: 2026年4月
**作者**: 文明（武汉大学信息管理学院博士生）

## 二、系统架构设计

### 2.1 总体架构

系统采用经典的分层架构设计，自下而上分为五个层次：

| 层次 | 名称 | 核心功能 | 技术实现 |
|------|------|----------|----------|
| L1 | 采集对象层 | 定义三层架构采集目标 | 104个平台预置数据 |
| L2 | 数据持久层 | 存储平台信息、采集记录、中间快照 | SQLite本地数据库 |
| L3 | 业务逻辑层 | 采集引擎、4E评分算法、任务调度 | Python类库 |
| L4 | API层 | RESTful接口、SSE实时推送 | Flask框架 |
| L5 | 展示层 | Web可视化界面、交互图表 | Bootstrap + Chart.js |

### 2.2 核心组件关系

```
采集对象(104平台) → 采集引擎 → 4E评分 → SQLite存储 → REST API → Web可视化
```

**关键设计决策**：
1. **SQLite本地存储**: 无需外部数据库服务器，便于复现和迁移
2. **SSE实时推送**: 采集进度通过Server-Sent Events实时推送至前端
3. **断点快照**: 每10个平台自动保存中间状态，支持断点续采
4. **4E框架内置**: 评估算法直接嵌入采集引擎，实现"采集即评估"

## 三、数据库设计

系统采用SQLite关系型数据库，包含6张核心数据表：

### 3.1 表结构说明

| 表名 | 记录数 | 说明 |
|------|--------|------|
| platforms | 104 | 三层架构平台基础信息（预置） |
| collection_tasks | N | 采集任务定义与状态 |
| collection_records | 100+ | 采集记录与4E评分结果（核心数据） |
| collection_snapshots | N | 采集中间状态快照 |
| analysis_results | N | 预计算分析结果 |
| collection_logs | N | 采集日志 |

### 3.2 核心表字段（collection_records）

采集记录表是本系统最核心的数据表，字段设计直接对应4E评估框架：

**基础信息字段**: task_id, platform_id, platform_code, platform_name, tier, region, status

**供给保障维度 (C1)**: dataset_count, format_types, has_api, has_bulk_download

**平台服务维度 (C2)**: response_time, has_https, has_search, has_download, has_visualization

**数据质量维度 (C3)**: has_update_info, has_metadata, has_feedback

**利用效果维度 (C4)**: has_register, has_preview, app_count

**综合评分**: score_c1, score_c2, score_c3, score_c4, overall_score

## 四、采集引擎设计

### 4.1 三层架构采集范围

| 层级 | 数量 | 采集方式 | URL配置 |
|------|------|----------|---------|
| Tier 1: 省级 | 31 | 直接访问 | 官方URL预置 |
| Tier 2: 副省级 | 13 | 直接访问 | 官方URL预置 |
| Tier 3: 地级市 | 60(抽样) | 规则推测 | data.{city}.gov.cn |

### 4.2 4E评分算法

系统内置4E框架评分算法，每个维度得分范围为[0,1]：

**C1 供给保障得分** = 数据集数量得分×0.4 + 格式多样性×0.2 + API支持×0.2 + 批量下载×0.2

**C2 平台服务得分** = 响应时间得分×0.2 + HTTPS×0.1 + 搜索×0.2 + 下载×0.2 + 可视化×0.2 + API×0.1

**C3 数据质量得分** = 更新信息×0.4 + 元数据×0.3 + 反馈渠道×0.3

**C4 利用效果得分** = 注册×0.3 + 预览×0.3 + 应用成果×0.4

**综合得分** = (C1 + C2 + C3 + C4) / 4

### 4.3 功能检测机制

系统通过HTML内容分析自动检测平台功能：

| 功能 | 检测关键词 | 示例 |
|------|-----------|------|
| 搜索功能 | 'search', '搜索', 'query' | 页面是否包含搜索框 |
| 下载功能 | 'download', '下载', 'export' | 是否支持数据下载 |
| API接口 | 'api', '接口', 'developer' | 是否提供开发者接口 |
| 用户注册 | 'register', '注册', 'signup' | 是否支持用户注册 |
| 数据预览 | 'preview', '预览', 'view' | 是否支持数据预览 |
| 可视化 | 'chart', '可视化', '图表' | 是否提供图表展示 |

## 七、系统创新点

1. **三层架构全覆盖**: 首次构建覆盖"省级-副省级-地级市"完整链条的采集系统，突破以往仅关注省级的局限

2. **4E框架技术化**: 将理论评估框架直接嵌入采集引擎，实现"采集即评估"的理论-技术融合

3. **实时可视化**: 采集过程与结果分析的实时联动，提升研究透明度和可重复性

4. **断点续采**: 支持大规模采集任务的中断恢复，适应网络不稳定环境

5. **零外部依赖**: 纯Python实现，SQLite存储，单文件数据库，便于学术复现

## 九、使用说明

### 9.1 环境要求
- Python 3.8+
- 无需外部数据库（SQLite内置）
- 无需外部Web服务器（Flask内置）

### 9.2 安装步骤
```bash
cd ogd_collector_system
pip install -r requirements.txt
python app.py
```

### 9.3 访问系统
浏览器打开 http://127.0.0.1:5000

### 9.4 创建采集任务
1. 进入"采集管理"页面
2. 点击"新建采集任务"
3. 选择采集类型（完整/省级/副省级/地级市）
4. 点击"开始采集"
5. 在实时监控面板观察进度

### 9.5 数据导出
点击"数据看板"页面的"导出CSV"按钮，可导出所有采集记录用于后续分析。
