自动化采集体系

定时调度机制
全量采集

每月 1 日凌晨 2:00 执行，覆盖全部 88 个平台，预计耗时 45-60 分钟。

增量更新

每周一、周四凌晨 3:00 执行，仅采集数据集数量有变化的平台。

健康检查

每日 6:00、12:00、18:00 三次探测，检测平台可访问性与响应时间。

异常告警

当平台连续 3 次不可达或响应时间 > 30s 时，自动记录异常并推送告警。

反爬对抗策略
请求频率控制

动态调整请求间隔（1-5s 随机延迟），模拟真人浏览行为。

User-Agent 轮换

维护 20+ 条真实浏览器 UA 池，每次请求随机选取。

IP 代理池

当触发访问限制时，自动切换代理 IP 重试，最大重试 3 次。

Cookie 会话保持

对于需要登录的平台，维护 Session 状态，支持验证码自动识别（OCR）。

# 三层架构数据采集引擎核心逻辑 class CollectorEngine: def __init__(self, platform_config): self.platforms = platform_config # 88 个平台配置 self.session = requests.Session() self.session.headers.update({'User-Agent': random.choice(UA_POOL)}) def collect_single(self, platform): # Step 1: HTTP 可达性探测 resp = self.session.get(platform.url, timeout=10) if resp.status_code != 200: return {'status': 'unreachable', 'code': resp.status_code} # Step 2: HTML 解析提取元数据 soup = BeautifulSoup(resp.text, 'html.parser') datasets = soup.select(platform.dataset_selector) # Step 3: 4E 维度评分计算 scores = self.calculate_4e_scores(soup, datasets) # Step 4: 结构化存储 self.save_to_db(platform, scores, datasets) return {'status': 'success', 'datasets': len(datasets)} def batch_collect(self): # 批量采集：88 个平台顺序执行 + 异常隔离 for p in self.platforms: try: result = self.collect_single(p) logger.info(f"{p.name}: {result['status']}") except Exception as e: logger.error(f"{p.name} failed: {e}") continue # 单平台失败不影响整体流程
