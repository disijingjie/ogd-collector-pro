七步采集流水线
平台探测与可达性检测

通过 DNS 解析 + HTTP HEAD 请求检测平台可访问性。记录响应时间、HTTP 状态码、HTTPS 支持情况。不可达平台标记异常并记录日志。

requests.head() socket.gethostbyname() timeout=10s
页面结构识别与解析策略选择

根据平台技术架构自动选择解析策略：静态 HTML（BeautifulSoup）/ 动态渲染（Selenium）/ API 接口（requests+json）。每个平台配置独立的 CSS Selector 或 XPath 规则。

BeautifulSoup4 Selenium WebDriver requests + json
数据集元数据提取

提取数据集标题、描述、发布部门、数据类别、数据格式、更新日期、下载量、浏览量等 15+ 字段。对分页列表进行递归遍历，确保全部数据集被抓取。

标题/描述/部门 格式/日期/下载量 分页递归
六步数据清洗与质量控制

Step 1 缺失值补全：对空字段进行默认值填充或标记；Step 2 异常值检测：基于箱线图法识别离群值；Step 3 格式标准化：统一日期格式、编码格式（UTF-8）；Step 4 重复去重：基于 URL+标题的 MD5 去重；Step 5 字段对齐：不同平台的字段映射到统一 schema；Step 6 编码统一：全量转 UTF-8，处理 BOM 头。

4E 维度自动评分

基于预定义的评分规则，对每个平台在 C1(供给保障)、C2(平台服务)、C3(数据质量)、C4(利用效果) 四个维度进行自动打分。C1-C3 基于客观指标计算，C4 基于下载量等代理指标。

C1 供给 C2 服务 C3 质量 C4 效果
结构化存储与索引

平台元数据存入 SQLite（platforms 表），采集记录存入 collection_records 表，运行日志存入 collection_logs 表。同时生成 CSV/JSON 备份文件。

SQLite CSV JSON
版本快照与血缘追踪

每次采集生成时间戳快照（collection_snapshots 表），记录数据来源、采集时间、解析规则版本号、采集脚本 Git commit hash，确保研究可复现。

Git 版本控制 时间戳快照 血缘追踪

# 采集引擎核心架构
class CollectorEngine:
  def __init__(self, config):
    self.platforms = config.platforms
    self.session = requests.Session()
    self.ua_pool = load_ua_pool()

  def detect_platform_type(self, url):
    # 检测平台技术架构
    resp = self.session.get(url)
    if "vue" in resp.text:
      return "dynamic"
    return "static"

  def collect(self, platform):
    p_type = self.detect_platform_type(platform.url)
    if p_type == "static":
      return self.parse_static(platform)
    else:
      return self.parse_dynamic(platform)
