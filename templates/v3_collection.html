数据归纳、分类与清洗流程

原始采集数据来自88个政府数据开放平台，各平台的元数据字段命名、格式、分类体系存在显著差异。本研究通过四步清洗流程将其转化为统一的标准化数据集。

Step 1: 字段映射
将各平台不同的字段名映射到统一标准。例如：
• "数据集名称" / "资源标题" / "name" → dataset_title
• "更新日期" / "发布时间" / "update_time" → update_date
• "提供部门" / "责任部门" / "dept" → department
Step 2: 分类体系统一
各平台数据集分类标签不一致（如"交通"vs"交通运输"），采用关键词匹配+人工校验统一为12个一级主题：交通、环境、教育、医疗、经济、社会、文化、科技、农业、公共安全、城市规划、其他。
Step 3: 格式标准化
数据格式字段统一映射：
• CSV/Excel/XLS/XLSX → 表格
• JSON/XML/API → 接口
• PDF/DOC → 文档
• SHP/GeoJSON → 地理
Step 4: 质量校验
完整性：必填字段缺失率<20%
一致性：省份-平台关联正确性100%
时效性：数据集更新时间在近2年内
可用性：下载链接可访问性检验

统一指标体系构建过程

清洗后的标准化元数据需要进一步转化为可量化评估的指标。本研究基于4E理论框架，将原始字段映射为16个二级指标，再聚合为4个一级维度。

经济性(Economy)指标构建
原始数据字段：平台维护状态、域名注册年限、服务器响应时间
→ 映射指标：
• C1-投入强度 = 数据集总量 / 平台运营年限
• C2-成本控制 = 开放格式占比（CSV/JSON/API视为低成本）
→ 数据来源：collection_records.dataset_count, format_types
效率性(Efficiency)指标构建
原始数据字段：数据集更新频率、下载量、API调用次数
→ 映射指标：
• C3-更新效率 = 近一年更新数据集占比
• C4-访问效率 = 平均响应时间倒数（标准化）
→ 数据来源：collection_records.response_time, 数据集update_date
有效性(Effectiveness)指标构建
原始数据字段：功能完整性（API/下载/搜索/可视化/反馈）
→ 映射指标：
• C5-功能完整度 = has_api + has_download + has_search + has_visualization + has_feedback（5项求和标准化）
• C6-数据覆盖度 = 主题分类覆盖数 / 12
→ 数据来源：collection_records各功能布尔字段
公平性(Equity)指标构建
原始数据字段：地区经济发展水平、平台层级、数据集可获取性
→ 映射指标：
• C7-区域均衡 = 该平台绩效与全国均值的偏差（越小越公平）
• C8-层级覆盖 = 省/市两级平台覆盖率
→ 数据来源：platforms.tier, collection_records.overall_score
指标计算示例：功能完整度(C5)
以山东省数据开放平台为例：
has_api=1, has_download=1, has_search=1, has_visualization=1, has_feedback=0
C5 = (1+1+1+1+0) / 5 = 0.80
经过Min-Max标准化后：C5_std = (0.80 - 0.20) / (1.00 - 0.20) = 0.75
该指标最终进入TOPSIS和DEA的输入/输出矩阵。

原始采集数据来自88个政府数据开放平台，各平台的元数据字段命名、格式、分类体系存在显著差异。本研究通过四步清洗流程将其转化为统一的标准化数据集。

Step 1: 字段映射
将各平台不同的字段名映射到统一标准。例如：
• "数据集名称" / "资源标题" / "name" → dataset_title
• "更新日期" / "发布时间" / "update_time" → update_date
• "提供部门" / "责任部门" / "dept" → department
Step 2: 分类体系统一
各平台数据集分类标签不一致（如"交通"vs"交通运输"），采用关键词匹配+人工校验统一为12个一级主题：交通、环境、教育、医疗、经济、社会、文化、科技、农业、公共安全、城市规划、其他。
Step 3: 格式标准化
数据格式字段统一映射：
• CSV/Excel/XLS/XLSX → 表格
• JSON/XML/API → 接口
• PDF/DOC → 文档
• SHP/GeoJSON → 地理
Step 4: 质量校验
完整性：必填字段缺失率<20%
一致性：省份-平台关联正确性100%
时效性：数据集更新时间在近2年内
可用性：下载链接可访问性检验

统一指标体系构建过程

清洗后的标准化元数据需要进一步转化为可量化评估的指标。本研究基于4E理论框架，将原始字段映射为16个二级指标，再聚合为4个一级维度。

经济性(Economy)指标构建
原始数据字段：平台维护状态、域名注册年限、服务器响应时间
→ 映射指标：
• C1-投入强度 = 数据集总量 / 平台运营年限
• C2-成本控制 = 开放格式占比（CSV/JSON/API视为低成本）
→ 数据来源：collection_records.dataset_count, format_types
效率性(Efficiency)指标构建
原始数据字段：数据集更新频率、下载量、API调用次数
→ 映射指标：
• C3-更新效率 = 近一年更新数据集占比
• C4-访问效率 = 平均响应时间倒数（标准化）
→ 数据来源：collection_records.response_time, 数据集update_date
有效性(Effectiveness)指标构建
原始数据字段：功能完整性（API/下载/搜索/可视化/反馈）
→ 映射指标：
• C5-功能完整度 = has_api + has_download + has_search + has_visualization + has_feedback（5项求和标准化）
• C6-数据覆盖度 = 主题分类覆盖数 / 12
→ 数据来源：collection_records各功能布尔字段
公平性(Equity)指标构建
原始数据字段：地区经济发展水平、平台层级、数据集可获取性
→ 映射指标：
• C7-区域均衡 = 该平台绩效与全国均值的偏差（越小越公平）
• C8-层级覆盖 = 省/市两级平台覆盖率
→ 数据来源：platforms.tier, collection_records.overall_score
指标计算示例：功能完整度(C5)
以山东省数据开放平台为例：
has_api=1, has_download=1, has_search=1, has_visualization=1, has_feedback=0
C5 = (1+1+1+1+0) / 5 = 0.80
经过Min-Max标准化后：C5_std = (0.80 - 0.20) / (1.00 - 0.20) = 0.75
该指标最终进入TOPSIS和DEA的输入/输出矩阵。

数据归纳、分类与清洗流程

原始采集数据来自88个政府数据开放平台，各平台的元数据字段命名、格式、分类体系存在显著差异。本研究通过四步清洗流程将其转化为统一的标准化数据集。

Step 1: 字段映射
将各平台不同的字段名映射到统一标准。例如：
• "数据集名称" / "资源标题" / "name" → dataset_title
• "更新日期" / "发布时间" / "update_time" → update_date
• "提供部门" / "责任部门" / "dept" → department
Step 2: 分类体系统一
各平台数据集分类标签不一致（如"交通"vs"交通运输"），采用关键词匹配+人工校验统一为12个一级主题：交通、环境、教育、医疗、经济、社会、文化、科技、农业、公共安全、城市规划、其他。
Step 3: 格式标准化
数据格式字段统一映射：
• CSV/Excel/XLS/XLSX → 表格
• JSON/XML/API → 接口
• PDF/DOC → 文档
• SHP/GeoJSON → 地理
Step 4: 质量校验
完整性：必填字段缺失率<20%
一致性：省份-平台关联正确性100%
时效性：数据集更新时间在近2年内
可用性：下载链接可访问性检验

统一指标体系构建过程

清洗后的标准化元数据需要进一步转化为可量化评估的指标。本研究基于4E理论框架，将原始字段映射为16个二级指标，再聚合为4个一级维度。

经济性(Economy)指标构建
原始数据字段：平台维护状态、域名注册年限、服务器响应时间
→ 映射指标：
• C1-投入强度 = 数据集总量 / 平台运营年限
• C2-成本控制 = 开放格式占比（CSV/JSON/API视为低成本）
→ 数据来源：collection_records.dataset_count, format_types
效率性(Efficiency)指标构建
原始数据字段：数据集更新频率、下载量、API调用次数
→ 映射指标：
• C3-更新效率 = 近一年更新数据集占比
• C4-访问效率 = 平均响应时间倒数（标准化）
→ 数据来源：collection_records.response_time, 数据集update_date
有效性(Effectiveness)指标构建
原始数据字段：功能完整性（API/下载/搜索/可视化/反馈）
→ 映射指标：
• C5-功能完整度 = has_api + has_download + has_search + has_visualization + has_feedback（5项求和标准化）
• C6-数据覆盖度 = 主题分类覆盖数 / 12
→ 数据来源：collection_records各功能布尔字段
公平性(Equity)指标构建
原始数据字段：地区经济发展水平、平台层级、数据集可获取性
→ 映射指标：
• C7-区域均衡 = 该平台绩效与全国均值的偏差（越小越公平）
• C8-层级覆盖 = 省/市两级平台覆盖率
→ 数据来源：platforms.tier, collection_records.overall_score
指标计算示例：功能完整度(C5)
以山东省数据开放平台为例：
has_api=1, has_download=1, has_search=1, has_visualization=1, has_feedback=0
C5 = (1+1+1+1+0) / 5 = 0.80
经过Min-Max标准化后：C5_std = (0.80 - 0.20) / (1.00 - 0.20) = 0.75
该指标最终进入TOPSIS和DEA的输入/输出矩阵。

七步采集流水线
Step 1: 种子URL发现
基于国务院《政务信息资源共享管理暂行办法》及各省数据局官网，建立初始URL种子库（88个平台入口）
Step 2: 页面类型识别
发送HEAD请求识别页面类型：静态HTML / 动态JS渲染 / REST API接口，自动选择采集策略
Step 3: 目录结构解析
解析平台目录树（主题分类 → 部门分类 → 数据集列表），建立层级映射关系
Step 4: 元数据提取
提取每条数据集的名称、描述、格式、标签、更新日期、下载量、提供部门等核心元数据字段
Step 5: 数据清洗标准化
统一日期格式（ISO 8601）、标准化部门名称（参照GB/T 2260）、格式映射（CSV/XLS/JSON/API）
Step 6: 质量校验入库
完整性校验（必填字段≥80%）、一致性校验（省份-平台关联正确）、重复性校验（URL去重）
Step 7: 增量更新维护
基于哈希值比对实现增量更新，仅采集变更数据，记录每次采集的完整日志
各省平台技术架构差异
静态页面型（约40%）
代表：山东、浙江、广东
技术：传统服务器端渲染，HTML+CSS
采集：Requests + BeautifulSoup直接解析
优势：响应快、结构稳定、易于解析
动态渲染型（约35%）
代表：北京、上海、四川
技术：Vue/React前端框架，异步加载

优势：交互丰富、用户体验好

数据归纳、分类与清洗流程

原始采集数据来自88个政府数据开放平台，各平台的元数据字段命名、格式、分类体系存在显著差异。本研究通过四步清洗流程将其转化为统一的标准化数据集。

Step 1: 字段映射
将各平台不同的字段名映射到统一标准。例如：
• "数据集名称" / "资源标题" / "name" → dataset_title
• "更新日期" / "发布时间" / "update_time" → update_date
• "提供部门" / "责任部门" / "dept" → department
Step 2: 分类体系统一
各平台数据集分类标签不一致（如"交通"vs"交通运输"），采用关键词匹配+人工校验统一为12个一级主题：交通、环境、教育、医疗、经济、社会、文化、科技、农业、公共安全、城市规划、其他。
Step 3: 格式标准化
数据格式字段统一映射：
• CSV/Excel/XLS/XLSX → 表格
• JSON/XML/API → 接口
• PDF/DOC → 文档
• SHP/GeoJSON → 地理
Step 4: 质量校验
完整性：必填字段缺失率<20%
一致性：省份-平台关联正确性100%
时效性：数据集更新时间在近2年内
可用性：下载链接可访问性检验

统一指标体系构建过程

清洗后的标准化元数据需要进一步转化为可量化评估的指标。本研究基于4E理论框架，将原始字段映射为16个二级指标，再聚合为4个一级维度。

经济性(Economy)指标构建
原始数据字段：平台维护状态、域名注册年限、服务器响应时间
→ 映射指标：
• C1-投入强度 = 数据集总量 / 平台运营年限
• C2-成本控制 = 开放格式占比（CSV/JSON/API视为低成本）
→ 数据来源：collection_records.dataset_count, format_types
效率性(Efficiency)指标构建
原始数据字段：数据集更新频率、下载量、API调用次数
→ 映射指标：
• C3-更新效率 = 近一年更新数据集占比
• C4-访问效率 = 平均响应时间倒数（标准化）
→ 数据来源：collection_records.response_time, 数据集update_date
有效性(Effectiveness)指标构建
原始数据字段：功能完整性（API/下载/搜索/可视化/反馈）
→ 映射指标：
• C5-功能完整度 = has_api + has_download + has_search + has_visualization + has_feedback（5项求和标准化）
• C6-数据覆盖度 = 主题分类覆盖数 / 12
→ 数据来源：collection_records各功能布尔字段
公平性(Equity)指标构建
原始数据字段：地区经济发展水平、平台层级、数据集可获取性
→ 映射指标：
• C7-区域均衡 = 该平台绩效与全国均值的偏差（越小越公平）
• C8-层级覆盖 = 省/市两级平台覆盖率
→ 数据来源：platforms.tier, collection_records.overall_score
指标计算示例：功能完整度(C5)
以山东省数据开放平台为例：
has_api=1, has_download=1, has_search=1, has_visualization=1, has_feedback=0
C5 = (1+1+1+1+0) / 5 = 0.80
经过Min-Max标准化后：C5_std = (0.80 - 0.20) / (1.00 - 0.20) = 0.75
该指标最终进入TOPSIS和DEA的输入/输出矩阵。
