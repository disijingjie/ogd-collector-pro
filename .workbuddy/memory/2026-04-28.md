## 腾讯云专家诊断结果

### 根本原因（已查明并修复）

问题链条：
1. 同步脚本ogd-sync.py全成功 → 文件确实更新到服务器（MD5一致）
2. 但v3_app.py中**缺少 `/collection` 路由定义** → 端口5001返回404
3. 用户浏览器看到"旧版本"实际是**浏览器缓存**（之前访问过5000端口的旧内容）
4. 同步脚本重启的是端口5000的systemd服务，**端口5001的v3 gunicorn从未被重启**

### 架构理解（关键）
- **端口5000**：旧版app.py（wsgi:app），由systemd管理
- **端口5001**：v3_app.py（v3_wsgi:app），**独立gunicorn进程**，需要单独重启
- **端口5002**：v4_app.py（v4_wsgi:app），独立gunicorn进程
- Nginx：/v3/ → 127.0.0.1:5001
- v3_collection.html已正确部署到 /opt/ogd-collector-pro/templates/

### 修复操作（分两步）

**第一步：添加 /collection 路由**
- 给v3_app.py添加 `/collection` 路由 → render_template('v3_collection.html')
- 正确重启端口5001 gunicorn（注意：PYTHONPATH不能和timeout混用，需用bash -c export）

**第二步：添加所有缺失的v3路由**
v3_app.py原有7个路由，新增7个：
- `/dashboard` → v3_dashboard.html
- `/platforms` → v3_platforms.html
- `/analysis` → v3_analysis.html
- `/collection_mechanism` → v3_collection_mechanism.html
- `/reproduce` → v3_reproduce.html
- `/research` → v3_research.html
- `/thesis` → v3_thesis.html

最终v3_app.py共14个路由，全部测试通过HTTP 200。

### 正确的gunicorn重启命令
```bash
bash -c "cd /opt/ogd-collector-pro && \
export PYTHONPATH=/home/ubuntu/.local/lib/python3.10/site-packages && \
/home/ubuntu/.local/bin/gunicorn v3_wsgi:app --bind 0.0.0.0:5001 --workers 2 --daemon"
```
**注意**：`PYTHONPATH=... timeout 5 cmd` 语法错误（timeout把PYTHONPATH当命令），必须用 `bash -c "export PYTHONPATH=... && cmd"`
