## 二、为什么之前看到"旧页面"

根本原因：**浏览器缓存** + **V8残留进程在5008端口**。

- nginx配置始终指向5000端口（V6）
- 但V8在5008端口也跑着，如果你直接访问 `:5008` 就会看到旧版
- 更常见的是**浏览器缓存了旧的CSS/JS**

**解决办法**：按 `Ctrl + F5` 强制刷新，或清空浏览器缓存。

## 三、SSH工具推荐（本地管理用）

你有三个选择，按推荐程度排序：

### 方案A：Windows Terminal + 系统OpenSSH（推荐，免费）

Windows 10/11 自带 OpenSSH，只需安装 Windows Terminal（微软商店免费下载）。

**优点**：
- 完全免费，系统原生支持
- 可以保存多个服务器连接配置
- 支持标签页、分屏

**连接命令**：
```bash
ssh ubuntu@106.53.188.187
# 免密登录已配置，直接进
```

**保存配置**：在 Windows Terminal 设置里添加一个配置文件：
```json
{
    "name": "OGD-Server",
    "commandline": "ssh ubuntu@106.53.188.187",
    "icon": ""
}
```

### 方案B：Termius（图形界面，跨平台）

官网：https://termius.com/  有免费版。

**优点**：
- 图形界面，不用记命令
- 可以保存主机、密码、密钥
- 有SFTP文件传输（直接拖拽上传）
- 手机App也能用

**配置**：
1. 下载安装 → 注册账号
2. 点击"New Host"
3. Address: `106.53.188.187`
4. Username: `ubuntu`
5. SSH密钥：导入 `C:\Users\MI\.ssh\id_ed25519`

### 方案C：MobaXterm（功能最强）

官网：https://mobaxterm.mobatek.net/

**优点**：自带SFTP、X11转发、多标签、宏录制

## 四、核心工作流：本地修改 → Git提交 → 服务器拉取

> 这是最重要的部分。以后所有修改都按这个流程走，**不需要再问我**。

### 4.1 目录对应关系

| 本地路径 | 服务器路径 |
|:---|:---|
| `C:\Users\MI\WorkBuddy\newbbbb\ogd_collector_system\v6_app.py` | `/opt/ogd-collector-pro/v6_app.py` |
| `C:\Users\MI\WorkBuddy\newbbbb\ogd_collector_system\templates\v6_*.html` | `/opt/ogd-collector-pro/templates/v6_*.html` |
| `C:\Users\MI\WorkBuddy\newbbbb\ogd_collector_system\static\*` | `/opt/ogd-collector-pro/static/*` |

### 4.2 修改流程（以修改 /research 页面为例）

**Step 1：在本地修改文件**

用 VS Code 打开：
```
C:\Users\MI\WorkBuddy\newbbbb\ogd_collector_system\templates\v6_research.html
```

改完后保存。

**Step 2：本地测试（可选但推荐）**

```bash
# 在PowerShell中
cd C:\Users\MI\WorkBuddy\newbbbb\ogd_collector_system
python v6_app.py
# 打开浏览器访问 http://localhost:5000/research
```

**Step 3：Git提交**

```bash
# 在PowerShell中
cd C:\Users\MI\WorkBuddy\newbbbb\ogd_collector_system
git add .
git commit -m "修改research页面：xxxx"
git push origin main
```

**Step 4：服务器拉取并重启**

通过SSH登录服务器后执行：
```bash
cd /opt/ogd-collector-pro
git pull origin main
sudo systemctl restart ogd-collector
```

或者，如果你配置了一键脚本，双击桌面上的 `ogd-update.bat` 即可。

### 4.3 常用Git命令速查

```bash
# 查看修改了哪些文件
git status

# 查看具体修改内容
git diff

# 提交所有修改
git add . && git commit -m "描述修改内容"

# 推送到远程
git push origin main

# 查看提交历史
git log --oneline -10
```

## 五、服务器管理命令速查

通过SSH登录服务器（`ssh ubuntu@106.53.188.187`）后可用：

```bash
# 查看V6服务状态
sudo systemctl status ogd-collector --no-pager

# 重启V6服务
sudo systemctl restart ogd-collector

# 查看实时日志
sudo journalctl -u ogd-collector -f

# 查看5000端口是否在监听
sudo ss -tlnp | grep 5000

# 检查是否有其他版本在跑
ps aux | grep -E "v3_app|v4_app|v8_app" | grep -v grep

# 测试页面内容（在服务器上）
curl -s http://127.0.0.1:5000/research | grep -o "陈传夫学术思想"
```

## 八、紧急回滚

如果某次修改搞坏了页面：

```bash
# 在服务器上
cd /opt/ogd-collector-pro
git log --oneline -5          # 查看历史提交
git reset --hard HEAD~1       # 回退到上一次提交
sudo systemctl restart ogd-collector
```
